# Инструкции Lamoda

## Область применения

- Lamoda работает в guest-режиме; `auth_domain_not_in_allowlist` означает только отсутствие SberId-preauth и не является причиной останавливать задачу.
- Для Lamoda основной поддерживаемый payment boundary - `bank_card_form`: дойти до формы ввода банковской карты и остановиться до ввода платежных данных.
- `bank_card_form` для Lamoda переопределяет default `sberpay` boundary: не требуй SberPay evidence и не останавливайся только потому, что SberPay нет среди способов оплаты.
- Не используй отдельный smoke-mode. Товар, вариант, адрес, контакты и способ оплаты проверяй как часть обычного сценария покупки.

## Поиск и выбор товара

- Выдели product identity и constraints из task, metadata и latest user reply: категория, бренд, цвет, размер, пол, бюджет, допустимые аналоги.
- Ищи товар через UI поиска Lamoda или через публичную страницу поиска Lamoda. В поисковый запрос включай product identity из текущей задачи; не hardcode SKU или product URL.
- Выбирай товар только из фактических результатов поиска и проверь соответствие перед добавлением в корзину.
- Если в корзине или checkout уже есть товар, не считай его подходящим автоматически. Проверь, что он соответствует текущей задаче. Если не соответствует, вернись к поиску или очисти/замени товар обратимыми действиями; если не можешь сделать это надежно, верни `needs_user_input`.
- Если точного товара нет, выбирай допустимый аналог только когда task прямо разрешает аналоги. Иначе верни `needs_user_input` с одним вопросом.
- Если нужен размер или вариант, выбери доступный вариант до добавления в корзину. Если размер не указан и сайт требует выбор, верни `needs_user_input`.

## Корзина и checkout

- Перед checkout проверь, что в корзине ровно один целевой товар, quantity `1`, и он соответствует текущим constraints.
- Если checkout уже открыт, все равно проверь товар в summary. Не продолжай с non-matching товаром без явного разрешения в текущей задаче или latest user reply.
- Используй адрес доставки из metadata/profile/latest user reply. Если адрес отсутствует, неоднозначен или Lamoda требует уточнение подъезда/квартиры/способа доставки, верни `needs_user_input` с одним конкретным вопросом.
- Если в metadata указан courier, выбирай курьерскую доставку. Если доступен только пункт выдачи или нужен выбор пользователя, верни `needs_user_input`.
- Для пункта выдачи ищи по адресу/метро из latest user reply. Если выбранный ПВЗ отличается от указанного адреса, но выглядит ближайшим вариантом, верни `needs_user_input` для подтверждения вместо молчаливого продолжения.
- Контактные поля `Имя`, `Фамилия`, `Телефон`, `Email` всегда приводи к значениям из текущей задачи/metadata/latest user reply. Не доверяй уже заполненным старым значениям на странице.
- Контакты заполняй один раз и проверь значения через `attr`/видимое состояние один раз. Если значения совпали, не перезаполняй эти поля и переходи к оплате. Повторяй `fill` только для конкретного поля, которое пустое, не совпало или рядом с которым видна явная ошибка.
- Если Lamoda просит SMS-код для входа или подтверждения телефона, верни `needs_user_input` с просьбой прислать код. Не выдумывай код.

## SMS и подтверждение телефона

- Если Lamoda просит SMS-код для входа или подтверждения телефона, верни `needs_user_input` с просьбой прислать код.
- Когда пользователь прислал код, вводи его только через `otp-fill`, например:

```bash
python /app/tools/cdp_tool.py --endpoint http://browser:9223 otp-fill \
  --selector 'input[autocomplete="one-time-code"]' \
  --code "$LATEST_USER_REPLY" \
  --digits 4 \
  --settle-ms 4000
```

- Не нажимай `Enter`, не нажимай `Получить код еще раз`, не экспериментируй с submit/resend и не делай прямых JS/Vue-мутаций.
- Если `otp-fill` вернул `accepted=true`, продолжай checkout.
- После `otp-fill accepted=true`, если checkout остался открыт и видна кнопка `Перейти к оплате`, следующим state-changing действием нажми `Перейти к оплате`. Не возвращайся к глубокой проверке radio/HTML и не перезаполняй контакты, если на странице нет видимой ошибки.
- Если `otp-fill` вернул `invalid_code=true`, верни `needs_user_input` и попроси новый 4-значный SMS-код.
- Если `otp-fill` вернул `still_open=true` без явной ошибки, не повторяй код и не пробуй альтернативные кнопки. Верни `needs_user_input` или `failed` с точным blocker: код введен один раз, но Lamoda не подтвердила шаг стандартным UI.

## Оплата банковской картой

- Для `bank_card_form` выбери `Сразу`, затем `Картой онлайн`.
- Не выбирай `СБП онлайн`, `Сплит`, `Долями`, сертификаты или рассрочку как замену банковской карте.
- Если клик по `Сразу` и `Картой онлайн` прошел без ошибки и в checkout body/snapshot видны эти тексты, считай способ оплаты выбранным достаточно для продолжения. Не доказывай выбранность через `input[type=radio]`, grep HTML или Vue/JS state.
- Не переключай `Сразу` и `Картой онлайн` повторно, если они уже выбраны/видны и нет пользовательской ошибки.
- Нажимай `Перейти к оплате` только после проверки товара, доставки, контактных полей и выбранного способа `Картой онлайн`.
- Если кнопка `Перейти к оплате` видна и не сообщает о disabled-состоянии, нажми ее. Если после клика остается checkout, проверь visible text на SMS, card form или конкретные ошибки; не застревай в повторной диагностике способов оплаты.
- После перехода к оплате остановись, когда видны поля карты, iframe/страница оплаты картой или текст вроде `Номер карты` / `Данные карты` / `Оплата картой`.
- На card boundary верни `status="needs_user_input"`, `order_id=null`, `payment_evidence=null` и короткое сообщение, что открыта форма оплаты картой и нужен номер карты. Не вводи платежные данные.
- Если доступен только СБП/Сплит/Долями или невозможно выбрать `Картой онлайн`, не продолжай. Верни `failed` или `needs_user_input` с точной причиной.
